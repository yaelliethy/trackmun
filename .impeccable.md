# TrackMUN — Design Context

## Users

**Delegates** are university students — generally tech-literate but not developers. They use the platform to register for conferences, view their assignment, check payment status, and access QR codes. Speed and clarity matter: they're often checking during conference breaks on mobile.

**Admins** are organizational staff — non-technical. They spend extended time managing delegates, approving registrations, configuring conference settings, and tracking attendance. The admin interface must prioritize scanability, fast actions, and reduced cognitive load.

**OC (Organizing Committee) & AC (Academic Committee / Chairs)** are staff members who interact with the platform intermittently — scanning attendance, assigning awards. Their interface should be straightforward and task-oriented.

## Brand Personality

**Formal · Modern · Trustworthy**

The platform represents a **Model United Nations conference** — a simulation of international diplomacy. It should project the seriousness and credibility of a real institution while feeling contemporary and efficient. No whimsy, no playfulness.

## Emotional Goals

- **Authority & Prestige** — like a real international organization
- **Calm Professionalism** — supports focus, doesn't distract
- **Modern Efficiency** — fast, responsive, well-structured
- **Subtle Energy** — restrained moments of dynamism during conference events (sessions, voting)

## Aesthetic Direction

**Primary: Minimal & Editorial**
- Strong, intentional typography with clear hierarchy
- Generous whitespace — content breathes
- Clean, structured layouts with precise alignment

**Secondary influence: Bold & Structured**
- Clear information hierarchy for data-dense views (tables, dashboards)
- Strong vertical rhythm and consistent spacing
- Deliberate use of weight and size to direct attention

## Visual References

| Reference | What to learn from it |
|-----------|----------------------|
| **Linear** | Clean layout, strong hierarchy, fast-feeling UI |
| **Vercel** | Minimal, modern, excellent whitespace and typography |
| **Stripe Dashboard** | Structured data presentation, polished components |
| **Notion** | Simplicity, readability, flexibility |

## Anti-References (Avoid)

- Playful, cartoonish, or "fun" UI (bright random colors, exaggerated rounding)
- Cluttered dashboards with too many widgets
- Outdated Bootstrap-style admin panels
- Overly flashy animations or gradients
- Social-media-heavy look (Instagram/Twitter vibes)
- Bounce/elastic easing, neon accents, glassmorphism
- AI slop: cyan-on-dark, purple-to-blue gradients, dark mode with glowing effects

## Color Palette

| Role | Value | Notes |
|------|-------|-------|
| **Primary** | `#04316c` (HSL 214 93% 22%) | Deep navy — authority, trust |
| **Primary Foreground** | `#ffffff` | White on primary surfaces |
| **Surface / Base** | `#ffffff` → near-white backgrounds | Light mode |
| **Neutral accents** | Muted grays tinted toward blue | For backgrounds, borders, text |
| **Hover/Active** | Muted blue variants derived from primary | Controlled, subtle |

**Avoid:** Bright/neon colors, multiple accent colors, anything that breaks the formal tone.

## Theme Mode

Both light and dark mode supported, with **system preference as default**.

## Design Principles

1. **Clarity over decoration** — Every element must earn its place. If it doesn't inform or enable action, remove it.
2. **Institutional confidence** — The UI should feel like it belongs to a serious organization, not a side project. Precision in spacing, alignment, and type conveys this.
3. **Calm efficiency** — Admin users live in the interface for hours. Minimize visual noise, maximize scanability, make common actions fast.
4. **Restrained dynamism** — Motion is purposeful and brief. Staggered reveals on page load, smooth transitions between states — never bouncy, never gratuitous.
5. **Typography-first hierarchy** — Let type weight, size, and spacing create structure. Don't lean on color or decoration to differentiate levels.
